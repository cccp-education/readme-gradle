== Objectif

Génération multilingue des README consommateur et plugin pour readme-gradle :
10 langues × 2 variantes = 20 fichiers Markdown dans deux répertoires dédiés.

== Périmètre

. `README.consommateurs/` — README destiné aux *utilisateurs* du plugin (API, tasks, config, quickstart).
. `README.plugin/` — README destiné aux *développeurs/contributeurs* du plugin (architecture interne, modules Kotlin, tests, pipeline CI).

Les deux répertoires vivent à la racine du repo (`foundry/public/readme-gradle/`), hors de `readme-plugin/`, pour séparer la documentation publique de la racine de travail.

Chaque fichier contient une ligne de navigation inter-langue :
[source]
----
🌐 Languages: [English](README.md) | [中文](README.zh.md) | [हिन्दी](README.hi.md) | [Español](README.es.md) | [Français](README.fr.md) | [العربية](README.ar.md) | [বাংলা](README.bn.md) | [Português](README.pt.md) | [Русский](README.ru.md) | [اردو](README.ur.md)
----

La langue courante est affichée en **gras** (Markdown `**...**`) plutôt qu'en lien, pour signaler la page active.

=== Placeholder `<YOUR_GITHUB_PAT>` préservé

Le placeholder `<YOUR_GITHUB_PAT>` apparaît dans les exemples de configuration YAML du README consommateur (section authentification Git/token). Il est *intact* dans les 10 fichiers de `README.consommateurs/` — aucune valeur réelle n'a été injectée. C'est cohérent avec :

. `readme-plugin/src/main/kotlin/readme/ReadmePlantUmlConfig.kt:73` — `token.takeIf { it.isNotBlank() && it != "<YOUR_GITHUB_PAT>" }`
. `readme-plugin/src/main/kotlin/readme/JGitRemoteValidator.kt:19` — guard `config.token == "<YOUR_GITHUB_PAT>"`
. `readme-plugin/src/main/kotlin/readme/ScaffoldTask.kt:154` — message `"replace <YOUR_GITHUB_PAT> with a real token"`
. Les features Cucumber (`2_scaffold.feature`, `3_process.feature`, `4_commit.feature`)
. Le test fonctionnel `ReadmeGradlePluginFunctionalTests.kt:39`

Le placeholder est le *sentinel* officiel du plugin pour distinguer "non configuré" de "configuré vide". À préserver lors de toute future évolution des README.

== Décisions techniques

. *Master source unique* : `README.md` (anglais) de chaque répertoire est la source de vérité. Les 9 autres fichiers sont déclarés traductions via le commentaire `<!-- translated from README.md rev 0.0.1 -->`. Toute évolution future doit partir de l'anglais.
. *Version figée à `0.0.1`* : les badges, la ligne de métadonnées et le marqueur `rev 0.0.1` référencent la version actuellement publiée sur Maven Central. À bumper synchroniquement lors d'une future release (RM-3 / v1.0.0).
. *Langues RTL sans balise `dir`* : pas de HTML `<div dir="rtl">` injecté. Le rendu Markdown GitHub gère nativement le sens des caractères Unicode (Arabic, Nastaliq). Les URLs/badges restent LTR car ce sont des URLs invariantes.
. *Switcher en bas d'en-tête* : la ligne `🌐 Languages: ...` est placée juste sous les badges/métadonnées, avant le premier `---`, pour visibilité immédiate sans alourdir le sommaire.
. *Pas de section "Translations" séparée* : chaque fichier est autonome, aucun index central des traductions. Le switcher fait office d'index.

== Sécurité

. Aucun token/PAT réel injecté — `<YOUR_GITHUB_PAT>` préservé partout (vérifié par `grep -rn "YOUR_GITHUB_PAT"` sur les 20 fichiers).
. Aucune clé API, mot de passe, ou secret dans les 20 fichiers.
. Badges shields.io : URLs publiques uniquement (Maven Central, Plugin Portal, GitHub Actions, License).
