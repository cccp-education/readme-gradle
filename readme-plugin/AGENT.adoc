== Projet

**Nom** : readme-gradle — plugin Gradle de génération README.adoc avec diagrammes PlantUML
**Stack** : Kotlin, Gradle (plugin development), JGit, PlantUML
**DAG** : N2
**Documentation** : AsciiDoc

=== Objectif

Générer des fichiers `README.adoc` à partir de sources `README_truth.adoc`/`README_truth_fr.adoc`,
en rendant les diagrammes PlantUML en PNG pour compatibilité GitHub.
Tâches : `generateReadme`, `transformReadme`, `commitGeneratedReadme`.

=== Non-Répétition des Opérations Coûteuses — Loi de l'Économie d'Encre

*PRINCIPE* : Une opération coûteuse (LLM, génération de README avec diagrammes PlantUML, rendu PNG) qui a déjà produit un résultat *valide et persistant* NE DOIT PAS être ré-exécutée sur la même entrée. Le résultat existant est la vérité ; le re-calcul est un gaspillage de tokens/crédits API/temps de build. L'IA n'est pas un générateur idempotent gratuit — c'est un service metered. Régénérer un README déjà produit et validé, re-rendre des PNG inchangés : chacun de ces gestes consomme du token sans produire de nouvelle connaissance. L'économie d'encre est la condition de viabilité économique du pipeline agentique MVP0.

*RÈGLE* :
. Avant toute opération coûteuse, **vérifier l'existence d'un artefact résultat** (`README.adoc` généré, `README_fr.adoc`, PNG de diagrammes, cache disque).
. Si l'artefact existe et couvre l'entrée demandée → **réutiliser sans ré-invoquer**.
. Seul le **delta** (sources `README_truth.adoc` modifiées, diagrammes manquants, blocs changés) doit être calculé — jamais l'ensemble.
. Ne **jamais écraser** un résultat validé par un re-calcul identique. L'écrasement est un symptôme de non-idempotence économique.
. L'invalidation est **explicite et ciblée** (fichier truth modifié, diagramme ajouté), jamais globale par défaut.

*ANTI-PATTERNS SPÉCIFIQUES README* :
. *Régénération README systématique* : re-appeler le LLM sur tout `README_truth.adoc` parce qu'un seul bloc a été édité.
. *Re-rendu PNG inconditionnel* : re-générer tous les PNG de diagrammes parce qu'un seul `.puml` a changé.
. *Cache-busting silencieux* : un `writeText()` qui écrase `README.adoc` généré avant même d'avoir lu l'existant.
. *Re-commit inutile* : `commitGeneratedReadme` qui re-commit un README identique au précédent.

*PROTECTION* :
. Lecture avant écriture : charger `README.adoc` existant avant de régénérer ; comparer hash de `README_truth.adoc` avec hash stocké en metadata du README généré.
. Delta blocs : ne régénérer que les blocs dont la source truth a changé (hash comparison par bloc).
. `DisableCachingByDefault` ne dispense pas de l'idempotence applicative — Gradle ne sait pas ce qu'est un "README déjà valide", seul le domaine le sait.

*RÉFÉRENCE* : Source canonique — `bakery-gradle/bakery-plugin/AGENT.adoc` section "Non-Répétition des Opérations Coûteuses — Loi de l'Économie d'Encre" (Session 153). Portée transverse : 10 boroughs LLM metered. Potentiel 50-80% d'économie LLM sur le pipeline MVP0 complet.

=== Composants Clés

* **ReadmePlugin** : Point d'entrée — enregistre extension + 3 tâches
* **ScaffoldTask** : Crée `readme-truth.yml` + `.github/workflows/readme_truth.yml`
* **ProcessReadmeTask** : Scan sources, rend PlantUML → PNG, réécrit blocs
* **CommitGeneratedReadmeTask** : JGit commit/push en CI
* **Cucumber** : 7 scénarios BDD (Minimal, Scaffold, Process, Commit, Integration)

=== Fin — 6 étapes

1. `.agents/sessions/{N}-{sujet}.adoc`
2. `PROMPT_REPRISE.adoc` (max 70 lignes)
3. `INDEX.adoc`
4. `SESSIONS_HISTORY.adoc`
5. `TEST_COVERAGE_ANALYSIS.adoc` (si tests modifiés)
