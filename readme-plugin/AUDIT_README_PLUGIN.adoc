== Anomalies Architecturales

=== 1. ScaffoldTask.kt — 294 lignes (28.8% du src/main)

`ScaffoldTask` est la plus grosse classe du plugin, concentrant :
création de répertoires, génération de `readme.yml`, génération de workflows
GitHub Actions, parsing YAML, gestion des erreurs.

*Recommandation* : Extraire la génération de workflows dans un délégué
`WorkflowGenerator` et le parsing YAML dans un service dédié.

=== 2. Dépendances compileOnly non documentées

Le `build.gradle.kts` déclare 10 dépendances en `compileOnly` (bundles `readme-ai`
et `readme-utils`) sans qu'aucun usage ne soit visible dans les 9 fichiers source.
Si ces dépendances sont réellement inutilisées, elles devraient être supprimées
pour alléger le build et éviter la confusion.

=== 3. 10 EPICs non démarrés — Backlog gelé

Après la code review (session 002), 10 EPICs restent « À FAIRE » sans aucune
priorisation ni découpage en user stories. Le projet est en suspens.

=== 4. Tests unitaires insuffisants

Ratio tests unitaires / source = 0.20x (201 lignes pour 1 022 lignes).
6 classes sur 9 n'ont *aucun* test unitaire. La couverture repose entièrement
sur les Cucumber (tests d'intégration lents).

=== 5. Pas de couverture de code

Absence totale de Kover ou tout autre outil de mesure de couverture.
Impossible de quantifier la qualité du code ou de détecter les régressions.

=== 6. plugin_snapshot.adoc — 2 913 lignes non gouvernées

Ce fichier massif à la racine du `readme-plugin/` n'est pas documenté dans
INDEX.adoc, n'a pas de raison d'être claire, et n'est pas versionné dans
la gouvernance (pas dans `.agents/`).
